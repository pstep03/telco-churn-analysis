# Telco Customer Churn Analysis

## Project Overview

This project analyzes customer churn behavior using a telecommunications dataset. The objective is to identify factors associated with customer churn and build predictive models that estimate the probability of a customer leaving the service.

The project demonstrates a typical data science workflow including SQL-based data exploration, data cleaning, exploratory data analysis, machine learning modeling, and statistical interpretation.


## Tools and Technologies

### SQL
- SQLite
- Table creation and schema design
- Business KPI queries
- Churn segmentation analysis

### Python
- Pandas

- Logistic Regression (GLM)

- Confidence intervals

### Environment
- Jupyter Notebook


## Project Workflow

1. **Data Cleaning (Python)**
   - Handle missing values
   - Convert data types
   - Create engineered features

2. **Database Setup (SQL)**
   - Create the customer dataset schema
   - Define customer attributes and churn indicator

3. **SQL Analysis**
   - Calculate overall churn rate
   - Analyze churn by contract type
   - Analyze churn by tenure group

4. **Exploratory Data Analysis**
   - Identify patterns related to churn
   - Explore relationships between customer attributes and churn

5. **Statistical Modeling (R)**
   - Logistic regression interpretation
   - Odds ratio analysis
   - Confidence intervals

6. **Machine Learning Models**
   - Logistic Regression
   - Decision Tree
   - Random Forest

7. **Model Evaluation**
   - Confusion Matrix
   - ROC Curve
   - AUC Score

8. **Feature Importance Analysis**
   - Identify the variables that most strongly influence churn
